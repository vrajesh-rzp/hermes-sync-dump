Details
Notes Length: Standard
* UI/UX Feedback on Payment Methods Display Ishan Nahata provided feedback on the UI structure, noting that while the structure is largely fine, elements like text sizes and alignment are causing issues when displayed side-by-side. They suggested clarifying the title related to platform fees to specify "0% platform fees" and indicating that it applies to UPI but can also be enabled for cards, wallets, and other payment methods. Ishan Nahata also mentioned that the UI for elements related to platform fees doesn't need to be complex.
* Concept Discussion: Single Plan with Top-ups Gopi Bhatnagar introduced concepts for a single plan, primarily focusing on "additional topups within the plan," which initially grants 0% platform fees on UPI only, with top-ups required to unlock the 0% fee for other methods. Pingal Kakati suggested disabling the other options to create a feeling that everything is already in a "cart," but Ishan Nahata raised a concern that if UPI is disabled, users might question why. Gopi Bhatnagar clarified that the intention was to avoid creating an impression that users can disable the UPI plan to get other payment methods.
* Plan Naming and Alternative Versions The speakers discussed the plan name, with Pingal Kakati suggesting to call it a "starter plan" instead of "UPI starter" since it can be made into a non-UPI plan. Ishan Nahata agreed that the name should be generic, suggesting 'starter' and using 'UPI' or 'all payment methods' as a descriptor/tagline. Gopi Bhatnagar presented a second concept, "top up with optimized savings," which is a different communication approach for the same plan, emphasizing savings like 17% or two months free for an annual subscription. Pingal Kakati questioned if the second version was necessary as a separate experiment, seeing it as an optimized version of the first concept.
* Selecting the Optimal Plan Concept for Experimentation The team decided to focus on the most optimized version of the single plan concept, specifically V2, which Ishan Nahata liked for its clean UI. Pingal Kakati stressed that the experiments should be larger, construct-level changes rather than minor UX adjustments, concluding that they should select the most promising concept and move forward with it. They also dismissed a concept where users would have too much control over individual payment methods, as it would create too many versions of subscriptions and management difficulties.
* Slider Functionality and Plan Ordering Gopi Bhatnagar mentioned having a final UI for a slider feature. Pingal Kakati criticized the plan ordering, noting a lack of logical sorting based on criteria like transaction amount or pricing, and emphasized that a logical list is necessary for user consumption. Ishan Nahata suggested that the recommendation should change as the user moves the slider, with the free plan recommended when the slider is to the left.
* Finalizing Design and Next Steps Gopi Bhatnagar, Ishan Nahata, and Pingal Kakati agreed to finalize the design for the most promising concept (likely V2/similar concepts like three and five) and have the engineering team start work by Monday. Pingal Kakati also stressed the importance of optimizing for consumption in the UI so that users know what they are getting in the plan. The speakers briefly discussed finding time to speak with Rajes about the plan for the next quarter.


Suggested next steps
* Pingal Kakati will finalize all three plans that will be implemented.
* Gopi Bhatnagar will take the feedback on the UI and work on it.
* Pingal Kakati will let engineering get started on the finalized plans.
* Pingal Kakati will discuss UI with Gopi Bhatnagar after Gopi Bhatnagar completes the UI changes.
* The group will find time to talk to Rajes about the plan for next quarter.
